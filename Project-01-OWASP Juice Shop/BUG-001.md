## 3. Description & Steps

### Bug Description
The global product search bar does not validate or sanitize user input. Searching for literal special characters (e.g., ".", ">") returns product list results instead of a "No results found" or "Invalid input" handling message.

### Steps to Reproduce
1. Navigate to the OWASP Juice Shop homepage.
2. Click on the Search bar at the top right.
3. Type the character `>` and press Enter.
4. Observe the results page.

| Expected Result | Actual Result |
| :---: | :---: |
| The system handles special characters gracefully by stripping them or displaying an explicit "No results found" notice. | The system attempts to execute the query, blindly displaying arbitrary search results or modifying the DOM behavior. |
