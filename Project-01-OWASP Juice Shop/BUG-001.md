
# Bug Report: Missing Input Validation on Search Bar Allows Arbitrary Special Characters

## 1. Bug Overview

| Bug ID | Severity | Priority | Component | Status |
| :---: | :---: | :---: | :---: | :---: |
| **BUG-001** | Medium | High | Global Search | Open |

## 2. Environment Details

| Environment | URL | Browser | Testing Date |
| :---: | :---: | :---: | :---: |
| QA / Production | https://juice-shop.herokuapp.com | Google Chrome Version 149.0.7827.116 | June 24, 2026 |

## 3. Description & Steps

### Bug Description
The global product search bar does not validate or sanitize user input. Searching for literal special characters (e.g., ".", ">") returns product list results instead of a "No results found" or "Invalid input" handling message.

### Steps to Reproduce
1. Navigate to the OWASP Juice Shop homepage.
2. Click on the Search bar at the top right.
3. Type the character `>` and press Enter.
4. Observe the results page.

| Expected Result | Actual Result |
| :---: | :---: |
|The system can process special characters without causing issues, either by filtering them out or displaying a "No results found" message. | The system displays unrelated search results instead of indicating that no matches were found. |
